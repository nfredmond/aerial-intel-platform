# Architecture

## Current direction

- Processing core: ODM/WebODM-compatible photogrammetry pipeline
- Delivery surface: web-first customer workspace with role-based access
- Auth backbone: Supabase Auth + organization memberships + entitlement gating

## Product positioning

DroneOps starts in planning/infrastructure workflows, but the architecture is designed for cross-profession customization (law, science, education, engineering, real estate, and other field-inspection domains).
