# Risk Register

_Last updated: 2026-04-19. This file tracks known risks carried by shipped surfaces of the Aerial Intel Platform. Each row names the risk, its blast radius, current likelihood, the control that holds it in place, and the accountable owner. Add a row when a surface ships; retire a row only when the underlying risk is structurally removed (not just dormant)._

| Risk | Impact | Likelihood | Mitigation | Owner |
|------|--------|------------|------------|-------|
| **Grounding bypass** — Aerial Copilot emits a sentence that reads plausible but has no `[fact:<id>]` token or cites an id not in the fact registry; user trusts it as cited. | Client-facing brief misstates an ODM output or job status; erodes the "every sentence cites a real fact" covenant promise. | Medium — happens on any prompt drift; mitigated but not impossible. | `validateGrounding` in `web/src/lib/copilot/grounding-validator.ts` drops unresolved sentences; skill refuses render if >30% drop. 31/31 copilot vitest suite pins behavior. First live run must be inspected by Nathaniel sentence-by-sentence before the skill is enabled for a second org. | nathaniel |
| **Spend runaway on multi-org rollout** — a second or third org enables copilot and hits the model in a loop (power user, misconfigured integration, or adversarial input) before quota catches it. | Cloud-bill surprise; provider-side rate limit hits and blocks legitimate Nat Ford traffic. | Low at current user count; Medium once >2 orgs are enabled. | Per-org integer-tenth-cent cap in `drone_org_ai_quota` with pre-check + post-call actual-spend record (`web/src/lib/copilot/quota.ts`); `AERIAL_COPILOT_ENABLED` env kill-switch takes precedence over per-org flag; `/admin/copilot` dashboard (Track A of the 2026-04-19 bridge slice) surfaces month-to-date spend before any org is enabled. | nathaniel |
| **Model deprecation or AI Gateway outage** — Anthropic rotates a model id, Vercel AI Gateway has a regional outage, or the `anthropic/claude-opus-4.7` / `anthropic/claude-haiku-4.5` slugs stop resolving. | Copilot surfaces return 500s or refuse; no client impact on core processing/delivery (copilot is opt-in, not load-bearing). | Medium over a 12-month horizon (model rotation is normal). | ADR-002 revised 2026-04-19 locked the provider-swap-by-env posture — `AI_GATEWAY_API_KEY` + string model ids mean swapping providers is a config change, not a code change. Model ids are centralized in `web/src/lib/copilot/pricing.ts`. Disclosure doc requires the model id shown inline on rendered output so any swap is visible to the user. | nathaniel |
| **Disclosure miscite on client deliverable** — a copilot-generated brief is copy-pasted into a client proposal without the AI-disclosure language from `docs/AI_DISCLOSURE.md`, or is edited by hand in a way that breaks a citation. | Violates Nat Ford Operating Covenant's "responsible AI use" + "truth without spin" gates; regulatory/professional exposure if cited conclusion is wrong. | Medium — human error scales with throughput. | `docs/AI_DISCLOSURE.md` names the required client-facing disclosure language per surface; brief download filename embeds the mission and model id; review covenant requires a human pass before any client delivery; share pages do not surface raw copilot output to external viewers (operator-initiated, authenticated only). | nathaniel |
| **TiTiler sidecar single-point-failure in prod** — the prod TiTiler instance goes down or is starved by NodeODM batch load if co-located; every orthomosaic viewer on every share page returns broken tiles. | All shipped Wave-1 raster viewer URLs break simultaneously; share-page links appear broken to clients. | Medium once TiTiler is live in prod (it is not yet — dev only). | ADR-002 Decision 5 commits to running TiTiler as its own service in prod (not co-located with NodeODM) behind a CDN that caches on `(url-hash, z, x, y)`. Until prod topology ships, the viewer renders an explicit fallback state when `AERIAL_TITILER_URL` is unset rather than a blank map. | nathaniel |
| **Processing-QA diagnosis confidently names a wrong root cause** — QA skill asserts "insufficient overlap" on a job that actually failed for a different reason, operator retries with the wrong mitigation. | Wasted compute + operator trust erodes; covenant's "defensible methods + citations" gate fails. | Medium — the rubric covers common ODM failure modes but not all; hallucinated certainty is the core failure mode of LLMs. | Grounding validator drops uncited claims; prompt explicitly asks the model to name uncertainty; panel is gated on `copilotRelevant` (only shown on actually-diagnosable jobs); disclosure copy on every output says "not a guarantee generated text is correct — only that each sentence cites a fact." First live run is on a synthetic or known-failure Toledo stub so the operator can grade the first diagnosis. | nathaniel |
